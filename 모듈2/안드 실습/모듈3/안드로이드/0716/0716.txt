UI스레드에서 어떤 특정 작업을 오래 할 경우 안드로이드 앱은 ANR(app not response)을 발생시키고 강제종료해버림

앱에서 뒤로가기를 했다가 다시 돌아오면 기존 정보가 다 날아감. 이를 해결하기 위해 service 사용

서비스는 스레드가 아님. 액티비티와 서비스 둘다 UI스레드에서 동작하는 컴포넌트임.

명심할 것은 UI 업데이트는 반드시 UI 스레드(메인 스레드)에서 해야한다는 것이다.

바인드 서비스 : 원격 서비스
프록시(proxy): 원격에 존재하는 서비스 객체의 함수 또는 메서드를 대신하여 호출하는 객체 -> 대리자
스텁(stub): 프록시에 의해 요청된 메서드 또는 함수 호출을 실제 구현된 객체에게 전송을 하는 역할을 함

서비스 객체를 구현하는 사람은 stub이라는 클래스를 상속하면 됨

홈버튼을 누르면 데이터가 남아있지만 뒤로가기 버튼을 누르면 데이터가 다 지워짐
