64bit, 32bit : 주소체계
32bit : 0 ~ (2의 32제곱-1)(번지) : 부여할 수 있는 주소의 개수 : 4GB 
: 하나의 주소가 32bit 의 크기를 가짐

java 파일명은 그 안에 있는 클래스명으로 하는 것이 규칙
클래스의 첫 문자는 대문자. 따라서 해당 java 파일명도 대문자

코드가 작성되어있는 .java 파일을 javac 라는 컴파일러로 컴파일하면
바이트코드로 변환된 .class 파일이 생성됨
그리고 java 라는 프로그램으로 JVM 을 구동시켜 바이트코드를 해석해서 .class 파일을 실행

JVM 은 플랫폼(OS+하드웨어(CPU)) 에 종속적. 그러나 자바 어플리케이션은 플랫폼에 독립적

리터럴 : 정수, 실수, 문자, 문자열, 논리(boolean)
boolean 형의 true, false 는 1, 0 이 아니라 그 자체가 데이터임
char 형은 unsigned 이기는 하지만 문자를 표현하기 위한 예외적인 것
자바는 그 외는 전부 signed 형이다 (부호가 있음)

byte < short < int < long < float < double
-표현범위
byte : -2^7 ~ 2^7-1
short : -2^15 ~ 2^15-1
char : 0 ~ 2^16-1(65535) : 유일한 unsigned 형
...

범정수확장(integer promotion) : 자동형변환
- 연산식에 있는 피연산자에 대하여 int형 보다 작은 자료형은 int 형으로 변환
- 보다 작은 자료형을 큰 자료형으로 형변환
- 연산식의 최종결과형은 연산식의 피연산자형임
- 데이터의 손실이 발생될 우려가 없는 상황이라면 컴파일러가 자동형변환시킴

char 는 음수를 표현할 수 없으므로 다른 타입의 데이터를 char 에는 절대 넣을 수 없다.

casting : 강제형변환
- 값의 손실 발생
- 부호의 손실
- 정밀도의 손실
