프로세스 : 메모리에 올라와있는 하나의 프로그램(실행하고 있는 프로그램의 작업공간)
스레드 : 프로세스내에서 진행되는 시작과 끝이 있는 일련의 작업
스레드풀 : 스레드를 담는 공간

멀티스레드는 동시성과 병렬성으로 실행됨
동시성 : 싱글코어 : 하나의 코어에서 여러개의 스레드가 번갈아가며 실행되는 성질
병렬성 : 멀티코어 : 여러개의 코어에서 개별 스레드를 동시에 실행하는 성질

임계영역 : 멀티스레드 프로그램에서 단 하나의 스레드만 실행할 수 있는 코드 영역
왜 임계영역이 필요한가? 하나의 스레드가 사용중인 공유객체를 다른 스레드가 변경할 수 없게 하기위해서 임계영역에서는 공유객체를 잠근다.
자바에서는 동기화 블록, 동기화 메소드로 구현함

start()호출 -> 스레드는 실행대기(Runnable 상태)가 됨 -> 스레드 스케쥴링에 의해 선택되면 run() 을 조금 실행(실행(execute)상태) -> 다시 실행대기 상태로 돌아가고 다른 스레드의 run() 조금 실행
-> 반복 : 멀티스레드이 동시성

때때로 스레드는 실행상태에서 일시정지상태(block)로 가기도 함. 이 때 다시 실행상태로 가기 위해서는 반드시 실행대기상태를 거쳐야함.
스레드 그룹 :  스레드 그룹의 장점으로는 그룹의 모든 스레드를 일괄 interrupt 할 수 있다는 점이다.
